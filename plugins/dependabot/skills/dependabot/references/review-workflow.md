# Review Workflow

Review all open Dependabot pull requests, categorize each by status, and generate an actionable execution plan report.

## Steps

### Step 1: Discover PRs

```bash
gh pr list --author "app/dependabot" --state open --json number,title,url,headRefName
```

### Step 2: Analyze Each PR

For each PR, use a subagent to determine:
- CI status (all checks passing, failing, or pending)
- Mergeable state (no conflicts)
- Risk assessment (low/medium/high based on dependency type and version jump)
- Recommendation (merge, fix, or close)

### Step 3: Categorize PRs

Assign each PR to exactly one category:
- **✅ MERGE** — CI passing, no conflicts, safe to merge immediately
- **🔧 FIX** — Fixable issues (lockfile sync, minor conflicts, etc.)
- **❌ CLOSE** — Superseded, duplicate, or should not be merged

### Step 4: Build Execution Plan

Order actions for sequential execution:
1. All MERGE actions first (lowest risk)
2. All CLOSE actions second (cleanup)
3. All FIX actions last (requires intervention)

### Step 5: Generate Report

Save to `docs/projects/dependabot/pr_review_YYYYMMDD_HHmmss.md` using the mandatory template below.

## Execution Plan

Execute actions sequentially. Update status after each action. Stop when encountering a FIX action.

| # | PR | Action | Status | Stop | Command | Notes |
|---|-----|--------|--------|------|---------|-------|
| 1 | #[n] | MERGE | ⏳ Pending | No | `gh pr merge [n] --squash --delete-branch` | |
| 2 | #[n] | CLOSE | ⏳ Pending | No | `gh pr close [n] -c "Superseded by #[x]"` | |
| 3 | #[n] | FIX | ⏳ Pending | **Yes** | See Fix Instructions below | Requires manual intervention |

### Status Legend
- ⏳ Pending — Not yet started
- 🔄 In Progress — Currently executing  
- ✅ Complete — Successfully finished
- ❌ Failed — Action failed, see Notes
- ⏸️ Blocked — Waiting on dependency or manual intervention

### Stop Conditions
- **Stop = Yes**: Halt execution and report. Manual intervention required.
- **Merge fails**: Halt execution, update status to ❌ Failed, add error to Notes.
- **CI fails post-merge**: Note in report for human review.
