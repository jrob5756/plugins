# Merge Workflow

Execute the Execution Plan from a Dependabot PR review report. Process each action sequentially, updating the report status after each action. Halt on FIX actions or any failure.

## Input

- **Report path** (required) — path to a `pr_review_YYYYMMDD_HHmmss.md` file generated by the review workflow

## Steps

### Step 1: Load Report

Read the review report file. Parse the Execution Plan table to identify all pending actions.

### Step 2: Update Progress Header

Update the Execution Progress section:
```markdown
**Status:** 🔄 In Progress  
**Current Step:** 1  
**Last Updated:** [current timestamp]
```

### Step 3: Execute Actions Sequentially

Loop through each row in the Execution Plan table in order:

```
FOR each row WHERE Status = ⏳ Pending:
    
    1. UPDATE Current Step in Execution Progress header
    
    2. CHECK Stop condition FIRST:
       IF Stop = "Yes" THEN:
           - Update row Status to ⏸️ Blocked
           - Update Notes: "Requires manual intervention"
           - GOTO Step 4 (Finalize)
    
    3. UPDATE row Status to 🔄 In Progress and SAVE report file
    
    4. EXECUTE the action:
       - Verify PR is ready (CI passing, no conflicts)
       - Execute the command (merge or close)
       - Confirm action completed successfully
    
    5. PROCESS result:
       IF success THEN:
           - Update row Status to ✅ Complete
           - Update Notes: "Completed [timestamp]"
           - SAVE the report file
           - CONTINUE to next row
       ELSE:
           - Update row Status to ❌ Failed
           - Update Notes: "[failure reason]"
           - SAVE the report file
           - GOTO Step 4 (Finalize)

END FOR
```

### Step 4: Finalize Report

When loop completes or halts:

1. Update Execution Progress:
   - **All complete**: Status = ✅ Complete
   - **Stopped on FIX**: Status = ⏸️ Blocked (waiting for fix)
   - **Failed**: Status = ❌ Failed
2. Update Last Updated timestamp
3. Save the report
4. Report summary to user

## Stop Conditions

| Condition | Action | Resume? |
|-----------|--------|---------|
| Row has `Stop = Yes` | Update to ⏸️ Blocked, halt | Yes, after fix |
| Merge command fails | Update to ❌ Failed, halt | Yes, after investigation |
| Close command fails | Update to ❌ Failed, halt | Yes, after investigation |
| All actions complete | Update progress to ✅ Complete | N/A |

## Example: Multi-PR Execution

**Starting state:**

| # | PR | Action | Status | Stop | Command | Notes |
|---|-----|--------|--------|------|---------|-------|
| 1 | #78 | MERGE | ⏳ Pending | No | `gh pr merge 78 --squash --delete-branch` | |
| 2 | #79 | MERGE | ⏳ Pending | No | `gh pr merge 79 --squash --delete-branch` | |
| 3 | #76 | CLOSE | ⏳ Pending | No | `gh pr close 76 -c "Superseded by #75"` | |
| 4 | #72 | FIX | ⏳ Pending | **Yes** | See Fix Instructions | Requires manual intervention |

**After merging #78 and #79, closing #76, stopping at #72:**

| # | PR | Action | Status | Stop | Command | Notes |
|---|-----|--------|--------|------|---------|-------|
| 1 | #78 | MERGE | ✅ Complete | No | `gh pr merge 78 --squash --delete-branch` | Completed 2025-12-16 10:32 UTC |
| 2 | #79 | MERGE | ✅ Complete | No | `gh pr merge 79 --squash --delete-branch` | Completed 2025-12-16 10:33 UTC |
| 3 | #76 | CLOSE | ✅ Complete | No | `gh pr close 76 -c "Superseded by #75"` | Completed 2025-12-16 10:33 UTC |
| 4 | #72 | FIX | ⏸️ Blocked | **Yes** | See Fix Instructions | Requires manual intervention |

**Execution Progress after halt:**
```markdown
**Status:** ⏸️ Blocked  
**Current Step:** 4  
**Last Updated:** December 16, 2025 at 10:33 UTC
**Reason:** Step 4 requires FIX action — run fix workflow for PR #72
```
